Стратегія функціонального тестування

Для тестування програми обрано метод "Чорної скриньки" (Black Box Testing) із застосуванням техніки Класів еквівалентності (Equivalence Partitioning).

Обгрунтування:
Оскільки вхідні дані (назви місяців) мають чітко визначений скінченний набір правильних значень, доцільно розділити всі можливі вхідні дані на класи:
1. Місяці з 31 днем (наприклад, January, March).
2. Місяці з 30 днями (наприклад, April, June).
3. Особливий випадок (February - 28 днів).
4. Невалідні дані (неіснуючі назви, числа, пусті рядки).

Такий підхід дозволяє перевірити коректність логіки програми без необхідності писати тест для кожного з 12 місяців окремо, забезпечуючи при цьому повне покриття функціоналу.
